How To Use GitHub?

GitHub is complex, but understanding a few basics will help you get started. In order to use GitHub, you’ll need to first be able to complete these few steps. How to use GitHub:

1. Sign up for GitHub

2. Install Git

3. Create a Repository

4. Create a Branch

5. Create and Commit Changes to a Branch

6. Open a Pull Request

7. Merge Your Pull Request

Step 1: Create An Account On gitHub

Create an acoount on gitHub https://github.com

Step 2: Download git for Desktop

Download Git for Desktop git https://git-scm.com/downloads

Step 3: git init

Open the VScode in a Repository and create a .git folder by using git init Command.

Step 4: Create Files

Make changes to the Project and check status by using git status Command

Step 5: git status

Shows the working tree status

Step 6: git add

Add file contents to the index

Step 7: git commit

Changes made with message to track Changes.

Step 8: gitHub Repository

Creat a Repository on a GitHub

Step 8-a: gitHub Repository

Provide details of repository on your account.

Step 8-b: gitHub Repository

Copy the URL of Repository in your account and use it to connect local repository with Remote one.

Step 8-c: gitHub Repository

Copy the URL of Repository in your account and use it to make connection between local repository and Remote.

Step 9: git Push

Push file to repository.

Step 10: git clone

Clone a repository.

Step 11: git pull

Fetch from and integrate with another repository or a local branch

Step 12: Create New Branch

In Git, branches are a part of your everyday development process. Git branches are effectively a pointer to a snapshot of your changes. When you want to add a new feature or fix a bug—no matter how big or how small—you spawn a new branch to encapsulate your changes. To Create New branch in the Existing Repository Use git checkout -b Branch_Name .

Step 14: Merge Conflict

A merge conflict is an event that takes place when Git is unable to automatically resolve differences in code between two commits. Git can merge the changes automatically only if the commits are on different lines or branches..
